# Step 1.5: 共通レイアウト設計

## このステップのゴール

ヘッダー・サイドバー・フッターに「何を置くか」を決める。

## 画面の構造

```
┌─────────────────────────────────────────┐
│              ヘッダー                    │
├────────┬────────────────────────────────┤
│        │                                │
│ サイド │       コンテンツエリア          │
│ バー   │       （各画面の中身）          │
│        │                                │
├────────┴────────────────────────────────┤
│              フッター                    │
└─────────────────────────────────────────┘
```

## 手順

### 手順 1: 必要なエリアを選ぶ

全てのエリアが必要なわけではない。アプリの規模で決める。

| アプリの規模 | 推奨構成 |
|-------------|---------|
| 画面数 1〜3 | ヘッダーのみ |
| 画面数 4〜10 | ヘッダー + 左サイドバー |
| 画面数 10以上 | ヘッダー + 左サイドバー + フッター |
| 補足情報が常に必要 | 上記 + 右サイドバー |

### 手順 2: ヘッダー vs サイドバーの使い分け

**判断基準：**
- 「自分」に関すること → **ヘッダー**
- 「仕事」に関すること → **サイドバー**

| 場所 | 置くもの | 具体例 |
|------|---------|--------|
| ヘッダー | ロゴ、自分の情報、通知、ログアウト | 「私」と「全体」に関すること |
| サイドバー | 業務メニュー、データへのリンク | 「仕事」と「機能」に関すること |

### 手順 3: 各エリアの内容を決める

#### ヘッダーに置くもの
| 位置 | 要素 |
|------|------|
| 左 | ロゴ / アプリ名 |
| 中央 | （通常は空） |
| 右 | 通知アイコン、ユーザーメニュー（名前、ログアウト） |

#### サイドバーに置くもの
- ナビゲーションメニュー（画面一覧）
- 現在の画面のハイライト表示
- 折りたたみ可能なカテゴリ

#### フッターに置くもの（使う場合）
- コピーライト
- 利用規約 / プライバシーポリシー
- バージョン情報

### 手順 4: ナビゲーション構造を整理する

Step 1 の画面一覧をグループ化してメニュー構造を作る。

**ポイント：**
- 詳細画面はメニューに出さない（一覧から遷移）
- グループ名は短く、分かりやすく
- 階層は2階層まで

```
📊 ダッシュボード
👥 利用者管理
   └ 一覧
   └ 新規登録
📝 記録
   └ 本日の予定
   └ 記録入力
👤 スタッフ
   └ スタッフ一覧
   └ シフト表
⚙️ 設定
```

## 成果物：共通レイアウト設計シート

以下の形式で `docs/ui/common-layout.md` を作成する。

```markdown
# 共通レイアウト設計

## 使用するエリア
- [x] ヘッダー
- [x] 左サイドバー
- [ ] 右サイドバー
- [ ] フッター

## ヘッダー
| 位置 | 要素 |
|------|------|
| 左 | ロゴ / アプリ名 |
| 右 | 通知アイコン、ユーザーメニュー（名前、ログアウト） |

## 左サイドバー（ナビゲーション構造）
- 📊 ダッシュボード
- 👥 利用者管理
  - 一覧
  - 新規登録
- 📝 記録
  - 本日の予定
  - 記録入力
- ⚙️ 設定
```

## チェックリスト

- [ ] 画面数に応じて、必要なエリアを決めたか？
- [ ] ヘッダーに置く要素を書き出したか？
- [ ] サイドバーのナビゲーション構造を作ったか？
- [ ] 詳細画面など「メニューに出さない画面」を区別したか？
- [ ] フッターが必要かどうか判断したか？

## 迷いやすい要素の判断

| 要素 | 判断 |
|------|------|
| 「自分の設定」 | ヘッダー（またはヘッダーのドロップダウン内） |
| 「アプリ全体の設定」 | サイドバー |
| 「ヘルプ」 | どちらでもOK。ヘッダー右端の「？」アイコンが定番 |
