# Step 4: ワイヤーフレーム作成

## このステップのゴール

ここまで決めたことを**「図」にして可視化**する。

## ワイヤーフレームとは

- 線と箱だけの設計図
- 色・フォント・装飾は省略
- 「何をどこに置くか」だけを示す

**なぜ必要か：**
1. 頭の中のイメージを「見える形」にできる
2. 実装前に「これで合ってる？」を確認できる
3. Claude Code に渡す「設計図」になる

## 作成手順

### 手順 1: 共通レイアウトの枠を描く

ヘッダー、サイドバーの外枠を描く。

```
┌─────────────────────────────────────────┐
│              ヘッダー                    │
├────────┬────────────────────────────────┤
│ サイド │                                │
│ バー   │       コンテンツエリア          │
│        │                                │
└────────┴────────────────────────────────┘
```

### 手順 2: コンテンツエリアを分割

Step 3 で決めたレイアウト型に沿って分割する。

### 手順 3: 各エリアに要素を配置

Step 2 で決めた情報グループを配置する。

### 手順 4: ラベルをつける

各要素に名前を書く。

## ワイヤーフレームの例

### 詳細画面（ヒーロー＋セクション型）

```
┌─────────────────────────────────────────┐
│ [ロゴ]                    [ユーザー] │
├────────┬────────────────────────────────┤
│ ナビ   │ ┌──────────────────────────┐ │
│        │ │ [写真] タイトル      [編集]│ │ ← ヒーロー
│ ・一覧 │ │        サブ情報           │ │
│ ・詳細 │ └──────────────────────────┘ │
│ ・登録 │ ┌──────────┐ ┌──────────┐ │
│        │ │ 基本情報  │ │ 連絡先   │ │ ← セクション
│        │ │ ・項目1  │ │ ・項目1  │ │
│        │ │ ・項目2  │ │ ・項目2  │ │
│        │ └──────────┘ └──────────┘ │
│        │ ┌──────────────────────────┐ │
│        │ │ 契約・利用情報           │ │ ← セクション
│        │ │ ・項目1  ・項目2         │ │
│        │ └──────────────────────────┘ │
│        │            [削除] [戻る] │ ← アクション
└────────┴────────────────────────────────┘
```

### 一覧画面（テーブル型）

```
┌─────────────────────────────────────────┐
│ [ロゴ]                    [ユーザー] │
├────────┬────────────────────────────────┤
│ ナビ   │ ページタイトル     [新規登録] │

│        │ └──────────────────────────┘ │
│        │ ┌──────────────────────────┐ │
│        │ │ 名前 │ 状態 │ 日付 │ 操作 │ │ ← テーブルヘッダ
│        │ ├──────────────────────────┤ │
│        │ │ ○○ │ 有効 │ 4/1 │ [詳細]│ │ ← データ行
│        │ │ ○○ │ 休止 │ 3/15│ [詳細]│ │
│        │ │ ... │ ... │ ... │ ...  │ │

│        │ ← 1 2 3 ... 10 →           │ ← ページネーション
└────────┴────────────────────────────────┘
```

## 作成ツール

| ツール | 特徴 | おすすめの場面 |
|-------|------|---------------|
| 紙とペン | 最速、修正も楽 | アイデア出し、初期段階 |
| Figma | 共有しやすい、再利用できる | チームで確認、きれいに残したい |
| Excalidraw | 手書き風、無料、シンプル | 1人で素早く、共有もしたい |
| テキスト（ASCII） | コードと一緒に管理できる | そのままClaude Codeに渡したい |

## Claude Code への渡し方

### 方法1: 画像として渡す

```
このワイヤーフレーム画像の通りに実装してください。
[画像を添付]

技術スタック: Next.js + shadcn/ui
```

### 方法2: テキストで構造を説明する

```markdown
## 利用者詳細画面

### 共通レイアウト
- ヘッダー: ロゴ（左）、ユーザーメニュー（右）
- 左サイドバー: ナビゲーションメニュー

### コンテンツエリア
1. ヒーローセクション（上部）
   - 左: 写真（丸型）
   - 中央: 利用者名（大）、サブ情報（小）
   - 右: [編集]ボタン

2. 情報カード（2列×2行のグリッド）
   - 基本情報カード: 生年月日、性別
   - 介護情報カード: 要介護度、認定期間
   - 連絡先カード: 住所、電話
   - 契約カード: 契約日、利用曜日

3. アクションエリア（下部）
   - [削除]ボタン、[一覧に戻る]ボタン
```

### 方法3: CLAUDE.md にパスを記載

```markdown
## UI設計ドキュメント

### ワイヤーフレーム
@docs/ui/wireframes/ ディレクトリを参照

### 実装ルール
- 上記ワイヤーフレームに従ってUIを実装すること
```

## 成果物

`docs/ui/wireframes/` に各画面のワイヤーフレームを保存する。

- 画像形式: `.png`, `.svg`
- テキスト形式: `.md`（ASCII図を含む）
