# Step 2: 情報設計

## このステップのゴール

「各画面に、何を表示し、何ができるようにするか」を決める。

## 手順

### 2-1. 書き出す：表示項目を全てリストアップ

各画面について、以下の2種類を**全て**書き出す。

| 種類 | 説明 | 例 |
|------|------|-----|
| 表示情報 | ユーザーが見るデータ | 名前、住所、ステータス、日付 |
| 操作 | ユーザーができるアクション | 編集ボタン、削除リンク、検索 |

**コツ：「これは書かなくてもいいかな」と思ったものも書く**

```
❌ 悪い例（省略しすぎ）
- 利用者名
- 連絡先
- ボタン

✅ 良い例（具体的に全部）
- 利用者ID（システムで自動採番）
- 利用者名（姓・名）
- ふりがな
- 生年月日
- 年齢（自動計算）
- 性別
- 郵便番号
- 住所（都道府県・市区町村・番地）
- 電話番号（自宅）
- 電話番号（携帯）
- 緊急連絡先（名前・続柄・電話番号）
- [編集]ボタン
- [削除]ボタン
- [一覧に戻る]リンク
```

### 2-2. グループ化：関連する項目をまとめる

書き出した項目を、**意味のある塊**に分ける。

**グループ化の3つの基準：**
1. 一緒に見る → 同時に必要な情報
2. 一緒に編集する → 同時に変更する情報
3. 同じ「話題」 → カテゴリが同じ

**グループ名のルール：**
- 名詞で、2〜4文字程度

```
✅ 良いグループ名
- 基本情報
- 連絡先
- 契約内容
- 利用状況

❌ 悪いグループ名
- 利用者の基本的な情報について
- その他の情報
- いろいろ
```

### 2-3. 優先順位：重要なものを上位に

グループができたら、**重要度の順番**をつける。

**優先順位の考え方：「この画面を開いて、最初に見たいのは何か？」**

| 優先度 | 説明 |
|--------|------|
| 高 | この画面の「主役」。開いた瞬間に見たい情報 |
| 中 | よく使う情報。主役の次に確認することが多い |
| 低 | たまに見る情報。あれば便利だが、常には不要 |

## 画面タイプ別のポイント

### 一覧画面
- 表示項目は「選ぶために必要な最小限」
- 詳細は詳細画面に任せる
- 詰め込みすぎない

### 詳細画面
- 表示項目は「知りたいこと全て」
- グループ化で整理
- 優先度低は隠してもOK（タブや折りたたみ）

### 入力フォーム
- 表示項目は「入力が必要なもの」
- 必須と任意を明確に
- 入力項目は最小限に

### ダッシュボード
- 表示項目は「全体を把握できる要約」
- 数値サマリー（件数、金額）
- 詳細へのリンクを忘れずに

## 成果物：画面情報設計シート

各画面について、以下の形式で `docs/ui/screens/[画面名].md` を作成する。

```markdown
# 利用者詳細画面

## 基本情報
- 画面タイプ: 詳細
- 主なユーザー: 全員

## 情報設計

### 優先度 高（常に表示）
【基本情報】
- 利用者名（姓・名）
- ふりがな
- 写真
- 生年月日 / 年齢
- 性別
- [編集]ボタン

### 優先度 中（通常表示）
【介護情報】
- 要介護度
- 認定有効期間

【契約・利用】
- 契約日
- 利用曜日
- 送迎の有無

【連絡先】
- 住所
- 電話番号
- 緊急連絡先

### 優先度 低（必要時のみ / 折りたたみ可）
【その他】
- 備考
- 登録日時
- 最終更新日時
- 利用者ID

【操作】
- [削除]ボタン
- [一覧に戻る]リンク
```
