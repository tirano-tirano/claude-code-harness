# Step 1: 画面洗い出し

## このステップのゴール

「このアプリには、全部で何画面必要か」を明確にする。

## 手順

### 1-1. ユーザーの種類を書き出す

まず、アプリを使う人（ユーザー）を明確にする。

```markdown
## ユーザー種別
- 管理者
- 一般スタッフ
- 閲覧のみユーザー
```

### 1-2. 各ユーザーの「やりたいこと」を動詞で書き出す

「〜したい」の形で書く。

```markdown
## 管理者のやりたいこと
- 利用者の一覧を見たい
- 利用者の詳細を見たい
- 新しい利用者を登録したい
- 利用者の情報を編集したい
- 今日の利用予定を確認したい
```

### 1-3. 「やりたいこと」を画面に変換する

| やりたいことの動詞 | 画面タイプ | 具体例 |
|------------------|-----------|--------|
| 一覧を見たい、探したい、選びたい | 一覧画面 | 利用者一覧、楽曲一覧 |
| 詳細を見たい、1件を確認したい | 詳細画面 | 利用者詳細、楽曲詳細 |
| 新しく作りたい、登録したい、追加したい | 入力フォーム | 利用者登録、楽曲追加 |
| 変更したい、編集したい、更新したい | 編集フォーム | （追加と共用も可） |
| 全体を把握したい、状況を俯瞰したい | ダッシュボード | 管理トップ、進捗概要 |
| 削除したい、確認したい（小さい操作） | ダイアログ | 削除確認、完了通知 |

## 成果物：画面一覧リスト

以下の形式で `docs/ui/screens.md` を作成する。

```markdown
# 画面一覧

## ユーザー種別
- 管理者
- スタッフ

## 画面一覧

| # | 画面名 | 画面タイプ | 主なユーザー | 概要 |
|---|--------|-----------|-------------|------|
| 1 | ダッシュボード | ダッシュボード | 全員 | 今日の状況を俯瞰 |
| 2 | 利用者一覧 | 一覧 | 全員 | 利用者を検索・選択 |
| 3 | 利用者詳細 | 詳細 | 全員 | 1人の利用者の全情報 |
| 4 | 利用者登録/編集 | フォーム | 管理者 | 利用者を追加・変更 |
| 5 | 本日の予定 | 一覧 | スタッフ | 今日の利用予定 |
| 6 | 記録入力 | フォーム | スタッフ | 介護記録の入力 |
```

## よくある失敗と対策

| 失敗 | 原因 | 対策 |
|------|------|------|
| 画面が足りない | 「やりたいこと」の洗い出しが甘い | ユーザー別に分けて考える |
| 画面が多すぎる | 1つの操作に1画面作りすぎ | 小さい操作はダイアログで済ませる |
| 後から画面が増える | 別のユーザーを忘れていた | 最初に「誰が使うか」を明確にする |
