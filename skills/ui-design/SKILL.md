# UI設計スキル

UIの設計から洗練されたデザインまで、全工程を支援するスキルです。

## 全体の流れ

UI設計は以下のステップで進めます：

```
Step 1:   画面洗い出し        → 何画面必要か
Step 1.5: 共通レイアウト設計  → ヘッダー・サイドバーの中身
Step 2:   情報設計            → 各画面に何を表示するか
Step 3:   レイアウト決定      → どのパターンで配置するか
Step 4:   ワイヤーフレーム    → 図にして可視化
Step 5:   ビジュアルデザイン  → 洗練された見た目に仕上げる
Step 6:   確認と修正          → スクリーンショットで確認し、問題を修正
```

**重要: Step 6 を省略しないこと。実装後は必ずスクリーンショットで確認する。**

## 各ステップの詳細

ユーザーの状況に応じて、適切なステップのリファレンスを読み込んでください。

### Step 1: 画面洗い出し
**いつ使う**: 新しいアプリの設計を始めるとき、必要な画面数を決めたいとき
**リファレンス**: `references/step1-screen-list.md` を読み込むこと

### Step 1.5: 共通レイアウト設計
**いつ使う**: ヘッダー・サイドバー・フッターの内容を決めたいとき
**リファレンス**: `references/step1.5-common-layout.md` を読み込むこと

### Step 2: 情報設計
**いつ使う**: 各画面に何を表示するか決めたいとき
**リファレンス**: `references/step2-information-design.md` を読み込むこと

### Step 3: レイアウト決定
**いつ使う**: 情報をどう配置するか決めたいとき
**リファレンス**: `references/step3-layout-patterns.md` を読み込むこと

### Step 4: ワイヤーフレーム作成
**いつ使う**: 設計を図にして可視化したいとき、実装に渡す設計図を作りたいとき
**リファレンス**: `references/step4-wireframe.md` を読み込むこと

### Step 5: ビジュアルデザイン
**いつ使う**: 「ダサい」を解消したいとき、洗練された見た目にしたいとき
**リファレンス**: `references/step5-visual-design.md` を読み込むこと

### Step 6: 確認と修正（必須）
**いつ使う**: 実装が完了したとき。**このステップは省略禁止**
**リファレンス**: `references/step6-review-and-fix.md` を読み込むこと

**やること**:
1. スクリーンショットを撮る（ライトモード・ダークモード両方）
2. チェックリストで確認する
3. 問題があれば修正する
4. 問題がなくなるまで繰り返す

## 使い方

### 最初から設計する場合
Step 1 から順番に進める。各ステップで成果物を作成し、次のステップに進む。

### 特定のステップだけ実行する場合
ユーザーが「レイアウトを決めたい」と言えば Step 3 のリファレンスを読み込む。
ユーザーが「デザインをよくしたい」と言えば Step 5 のリファレンスを読み込む。

### 成果物の保存先

UI設計の成果は、対応する feature ファイルの技術仕様 > UI セクションに記載する。

```
docs/features/{番号}_{機能名}.md の「## 技術仕様 > ### UI」セクション
```

画面一覧や共通レイアウトなど、複数機能にまたがるUI設計は `docs/architecture.md` に記載する。
ワイヤーフレーム画像などは `docs/ui/wireframes/` に保存し、feature ファイルからリンクする。

```
docs/
├── features/{機能名}.md   # 各機能の UI セクション
├── architecture.md        # 共通レイアウト、画面一覧
└── ui/wireframes/         # ワイヤーフレーム画像（必要な場合のみ）
    ├── [画面名].png
    └── ...
```

## 技術スタック（実装時）

- UIコンポーネント: shadcn/ui
- スタイリング: Tailwind CSS
- アイコン: lucide-react

## 【必須】実装時のデザインルール

**UIを実装する際は、以下のルールを必ず守ること。違反すると「ダサい」UIになる。**

### 1. 余白（最重要）

**余白は「多すぎるかな？」と思うくらいで丁度良い。窮屈なUIは絶対に避ける。**

```
【絶対に守る余白の最小値】

■ ページ全体
ページ左右パディング:     px-6 (24px) 以上
ページ上下パディング:     py-6 (24px) 以上

■ ヘッダー
ヘッダー高さ:             h-14 (56px) または h-16 (64px)
ヘッダー内パディング:     px-6 py-4
ヘッダーとコンテンツの間: 境界線(border-b)で区切る

■ サイドバー
サイドバー幅:             w-64 (256px) 以上 ← w-48 以下は禁止
サイドバー内パディング:   p-4 (16px) 以上
メニュー項目:             py-2 px-3 以上
メニュー項目間:           space-y-1 (4px)
セクション間:             mt-6 (24px) 以上
セクション見出し下:       mb-2 (8px)

■ コンテンツエリア
コンテンツ内パディング:   p-6 (24px) 以上

■ タブ
タブ内パディング:         px-4 py-2
タブ間:                   gap-1 (4px) 以上
タブとコンテンツの間:     mt-6 (24px) 以上 ← mt-2, mt-4 は禁止

■ 統計・サマリー表示（「残り 0 曲採用 0」のような表示）
統計項目間:               gap-4 (16px) 以上 または gap-6 (24px)
統計とコンテンツの間:     mb-6 (24px) 以上

■ カード
カード内パディング:       p-6 (24px) 以上 ← p-4 は禁止
カード間:                 gap-6 (24px) 以上

■ フォーム
フォーム項目間:           gap-6 (24px) 以上 ← gap-4 は禁止
セクション間:             gap-8 (32px) 以上

■ テーブル
行の高さ:                 py-3 以上
ヘッダー:                 bg-slate-800 + border-b で区別
チェックボックス列:       w-12 固定

【禁止事項】
❌ カードに p-2, p-3, p-4 を使う
❌ フォーム項目間に gap-2, gap-4 を使う
❌ タブとコンテンツの間に mt-2, mt-4 を使う
❌ サイドバー幅を w-48 以下にする
❌ 余白を省略して要素を詰め込む
```

### 2. 複雑なコンポーネントの設計原則

**多くの要素を含むUI（ツールバー、フィルター、プレイヤーなど）には以下の原則を適用する。**

```
【原則1: グループ化】
- 関連する要素をグループとしてまとめる
- グループ内: gap-2〜gap-3（狭い）
- グループ間: gap-6〜gap-8（広い）

【原則2: 優先順位】
- 重要なもの → 目立つ位置（左または中央）
- 補助的なもの → 端に配置
- 使用頻度が低いもの → 折りたたみ、ドロップダウン、または別の行

【原則3: ゾーン配置】
- グループを「ゾーン」として配置する
- ゾーンの数は内容に応じて決める（2〜4が一般的）
- 左寄せ: プライマリ操作
- 右寄せ: セカンダリ操作、設定
- 中央（必要なら）: 幅を取る要素

【原則4: 複数行】
- 1行に収まらない場合は無理に詰め込まない
- 2行構成、または展開可能なセクションを検討する

【原則5: 幅の統一】
- 横に並ぶ入力欄: w-16, w-20, w-24 など固定幅
- ドロップダウン: w-32, w-40 など固定幅
- ラベル: w-10, w-12 など固定幅
- 要素間: gap-4 で統一

【原則6: 最小サイズ】
- 1行構成のバー: h-14 (56px) 以上
- 2行構成のバー: h-20〜h-24 (80〜96px)
- パディング: px-6 py-4

【禁止事項】
❌ 1行に要素を詰め込みすぎる
❌ 入力欄やドロップダウンの幅を固定しない（ガタガタになる）
❌ グループ化せずに要素を並べる
```

### 3. 背景色と階層（視覚的区別）

**異なるエリアは異なる背景色で区別する。全部同じ色は禁止。**

```
【ライトモード】
ページ背景:     bg-slate-50 または bg-gray-50（真っ白 #FFF は禁止）
サイドバー:     bg-white または bg-slate-100
コンテンツ:     bg-white
カード:         bg-white + shadow-sm + border
ヘッダー:       bg-white + border-b

【ダークモード】
ページ背景:     bg-slate-950 または bg-zinc-950（真っ黒 #000 は禁止）
サイドバー:     bg-slate-900 または bg-zinc-900
コンテンツ:     bg-slate-900/50
カード:         bg-slate-800 + border-slate-700
ヘッダー:       bg-slate-900 + border-b border-slate-800

【禁止事項】
❌ 真っ白 (#FFFFFF, bg-white) をページ背景に使う
❌ 真っ黒 (#000000, bg-black) をページ背景に使う
❌ すべてのエリアを同じ背景色にする
❌ 境界線なしでエリアを区切る
```

### 3. 境界と区切り

**エリア間には必ず境界を設ける。**

```
【必須の境界】
ヘッダー下:           border-b border-slate-200 (dark: border-slate-800)
サイドバー右:         border-r border-slate-200 (dark: border-slate-800)
カード:               border border-slate-200 + rounded-lg + shadow-sm
セクション区切り:     border-t または gap-8 以上の余白

【サイドバーのセクション区切り】
- セクション見出し（素材、動画、設定など）は text-xs text-slate-500 uppercase tracking-wider mb-2
- セクション間は mt-6 以上の余白を取る
```

### 4. タブのスタイル

```
【タブの必須スタイル】
- 選択中: bg-slate-100 (dark: bg-slate-800) + font-medium
- 非選択: bg-transparent + text-slate-600 (dark: text-slate-400)
- タブ間: gap-1 以上
- タブ自体: px-4 py-2 rounded-md

shadcn/ui の Tabs コンポーネントを使用すること。
```

### 5. Empty State（空の状態）

**データがない時は、必ず以下を表示する：**

```
【Empty State の必須要素】
1. アイコン（大きめ、薄いグレー）: h-12 w-12 text-slate-400
2. メッセージ（分かりやすく）
3. サブメッセージ（何をすればいいか）
4. アクションボタン（あれば）

【レイアウト】
- 中央寄せ: flex flex-col items-center justify-center
- 最小高さ: min-h-[400px] または画面の中央
- 余白: gap-4 以上

【例】
<div class="flex flex-col items-center justify-center min-h-[400px] gap-4">
  <Music class="h-12 w-12 text-slate-400" />
  <div class="text-center">
    <p class="text-lg font-medium">選別する楽曲がありません</p>
    <p class="text-sm text-slate-500 mt-1">楽曲をアップロードして始めましょう</p>
  </div>
  <Button>楽曲をアップロード</Button>
</div>
```

### 6. 色

```
使用する色: 3色まで
- ベース: slate 系（slate-50 〜 slate-950）
- アクセント: 1色のみ（blue-600 など）
- 背景: slate-50（ライト）/ slate-950（ダーク）

禁止:
❌ 原色（#FF0000, #0000FF）
❌ 4色以上のアクセントカラー
❌ グラデーション
❌ 真っ白・真っ黒の背景
```

### 7. 統一感

```
角丸:     rounded-lg (8px) で統一
影:       shadow-sm のみ（shadow-lg 以上は禁止）
ボーダー: border-slate-200 (dark: border-slate-700/800)
```
