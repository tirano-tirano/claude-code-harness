# ブレインストーミング → ドキュメント出力

## 概要

考えたことを空中に消さない。ブレインストーミングの成果を必ずドキュメントに残す。

**基本原則：** ブレインストーミングのゴールは「話し合うこと」ではなく「判断可能な状態を作ること」。

## 4 つの Phase

```
Phase 1: 準備 → Phase 2: 対話 → Phase 3: 構造化 → Phase 4: ドキュメント出力
```

### Phase 1: 準備

1. **出力先を決める** — ブレスト結果をどのドキュメントにするか最初に決める
2. **メモファイルを作成** — `docs/journal/{date}-brainstorm-{topic}.md` に対話中のメモを記録

出力先の対応表：

| ブレストの目的 | 出力先ドキュメント |
|--------------|-------------------|
| 新機能の企画 | docs/prd/{feature-name}.md |
| 技術選定 | docs/adr/{number}-{title}.md |
| UI設計の方向性 | docs/ui/（ui-design スキルに引き継ぎ） |
| テスト戦略 | docs/test-plans/{feature-name}.md |
| バグの原因調査 | docs/journal/{date}.md |
| その他 | docs/journal/{date}-{topic}.md |

### Phase 2: 対話（ソクラテス式）

ブレインストーミングは一方的な提案ではなく、質問と対話で進める。

**対話のガイドライン：**

| やること | やらないこと |
|---------|------------|
| 1回に1つの質問をする | 5個の質問を一度に投げない |
| 相手の回答を受けて深掘りする | 自分の結論に誘導しない |
| 「なぜ？」「誰のため？」を聞く | 「こうすべき」と断定しない |
| 選択肢を提示して選んでもらう | 1つの案だけ提示しない |
| メモファイルにリアルタイムで記録 | 対話が終わってからまとめない |

**メモの書式：**

```markdown
## ブレインストーミングメモ — {トピック}
日時: {date}

### 議論ポイント
- Q: {質問}
  A: {回答・判断}
  → 決定 / 未決定 / 要調査

### 浮上したアイデア
- {アイデア1}: {メリット} / {懸念}
- {アイデア2}: {メリット} / {懸念}
```

### Phase 3: 構造化

対話が一区切りついたら、メモを以下のカテゴリに分類する：

| カテゴリ | 内容 | 次のアクション |
|---------|------|--------------|
| ✅ 決定事項 | 合意が取れた内容 | ドキュメントに反映 |
| ❓ 未決事項 | まだ決まっていない内容 | ドキュメントの「未決事項」セクションに記載 |
| 🔧 技術的な検討事項 | 技術選定や設計判断 | ADR として記録 |
| 🎨 UI/UX の検討事項 | 画面や操作の方向性 | ui-design スキルに引き継ぎ |

### Phase 4: ドキュメント出力

Phase 1 で決めた出力先に、構造化した内容を書き出す。

**PRD を出力する場合のテンプレート：**

```markdown
# {機能名} — 製品要求仕様書

## 背景
（Phase 2 の対話から抽出）

## 目的
（「なぜ作るのか」への回答）

## ユーザーストーリー
（「誰のためか」への回答から導出）

## 機能要件
（決定事項から抽出）

## 非機能要件
（技術的検討事項から抽出）

## 未決事項
（未決事項をそのまま記載 — 隠さない）

## 次のステップ
- [ ] 未決事項の解消
- [ ] API仕様の作成（document-lifecycle スキル）
- [ ] UI設計（ui-design スキル）
- [ ] テスト計画（web-testing スキル）
```

## Red Flags — 以下に該当したら立ち止まること

| Red Flag | 正しい対応 |
|----------|-----------|
| 「メモを取らずに対話だけ進めた」 | 対話中にメモファイルに記録する。記憶は消える。 |
| 「ブレスト結果をドキュメントにしなかった」 | Phase 4 を実行する。ドキュメントにならないブレストは無かったのと同じ。 |
| 「結論が出なかったからドキュメントに書けない」 | 未決事項としてドキュメントに書く。「決まっていない」ことも記録すべき情報。 |
| 「1人で考えて答えを出した」 | ソクラテス式対話に戻る。質問して相手に考えてもらう。 |
| 「提案を5個並べて選ばせた」 | 1つずつ質問で深掘りする。選択肢の羅列は対話ではない。 |
| 「PRDのフォーマットに合わないから書かない」 | フォーマットは調整してよい。内容が残ることが重要。 |
